# 2026-03-13

## 添加多核 CPU 并行调度支持

### 变更内容
1. **docker-compose.yaml** — 新增 `OLLAMA_MAX_QUEUE=512`，增强 `OLLAMA_NUM_PARALLEL` 注释说明多核调度原理
2. **docker-compose.yaml.template** — 同步添加 `OLLAMA_MAX_QUEUE` 变量
3. **ollama.sh `do_optimize`** — `OLLAMA_NUM_PARALLEL` 计算逻辑升级：综合 VRAM 和 CPU 核心数取较小值（之前仅基于 VRAM），新增 `OLLAMA_MAX_QUEUE` 自动计算（NUM_PARALLEL × 64，范围 128~1024）
4. **ollama.sh `do_status`** — 新增「⚡ 并行调度配置」面板，展示 CPU 核心数、容器 CPU 限制、NUM_PARALLEL、MAX_QUEUE、MAX_LOADED_MODELS、KEEP_ALIVE、CONTEXT_LENGTH 等配置及活跃模型数
5. **ollama.sh `do_health`** — 新增第6项「并行调度」健康检查，检测 NUM_PARALLEL 是否已配置且大于1，总检查项从 9 增至 10
6. **ollama.sh `do_init`** — 默认配置和 .env 模板中添加 `OLLAMA_MAX_QUEUE=512`
7. **README.md** — 新增「多核 CPU 并行调度」章节，更新核心特性、优化参数表格、health 检查项数
